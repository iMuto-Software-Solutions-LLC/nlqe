# Query Engine Fixtures

This directory contains sample data and example patterns for the Query Engine prototype.

## Sample Data Files

### Parquet Files

The following parquet files are generated by `create_sample_data.py`:

#### `transactions.parquet` (2,500 rows)
Main fact table containing e-commerce transactions.

**Columns:**
- `transaction_id`: Unique identifier (INT)
- `date`: Transaction date (DATETIME)
- `customer_id`: Reference to customer (INT)
- `region_id`: Reference to region (INT)
- `product_name`: Name of product (STRING)
- `category`: Product category (STRING)
- `amount`: Transaction amount in dollars (FLOAT)
- `quantity`: Units purchased (INT)
- `is_return`: Whether this is a return (BOOL)
- `profit_margin`: Profit on transaction (FLOAT)

**Key Statistics:**
- Date range: 2024-01-01 to 2024-03-31 (Q1 2024)
- Categories: Electronics, Clothing, Food, Home, Sports
- Amount range: $5 to $2,000
- Return rate: ~10%

**Queries to try:**
- Simple: "How many transactions total?" → COUNT(*)
- Aggregation: "What is the total revenue?" → SUM(amount)
- Filtering: "Show me all Electronics transactions" → WHERE category = 'Electronics'
- Grouping: "Revenue by category?" → GROUP BY category
- Complex: "Return rate by category?" → CASE WHEN with GROUP BY

## Example Query Patterns

### `example_queries.yaml`

Contains 25+ example query patterns organized by category and difficulty level.

**Structure:**
```yaml
examples:
  - id: "unique_id"
    category: "aggregation|filtering|grouping|join|complex"
    difficulty: "easy|medium|hard"
    description: "Human readable description"
    question: "Natural language question"
    sql: "Expected SQL query"
    explanation: "Why this SQL works"
    tags: ["tag1", "tag2"]
```

**Categories:**

1. **Aggregation** (COUNT, SUM, AVG, MIN, MAX)
   - `simple_count`: Basic COUNT(*)
   - `simple_sum`: Basic SUM aggregation
   - `simple_avg`: Average calculation

2. **Filtering** (WHERE clauses)
   - `filter_by_category`: String comparison
   - `filter_by_date_range`: Date range filtering
   - `filter_by_amount`: Numeric comparison

3. **Grouping** (GROUP BY)
   - `group_by_category`: Single column GROUP BY
   - `group_by_with_count`: GROUP BY with aggregation
   - `group_by_region_customer`: Multi-column GROUP BY with JOIN

4. **Sorting** (ORDER BY, LIMIT)
   - `top_n_results`: Get largest N values
   - `bottom_n_results`: Get smallest N values

5. **Joins** (INNER, LEFT, multi-table)
   - `simple_inner_join`: Basic INNER JOIN
   - `multi_table_join`: Multiple JOINs
   - `left_join_example`: LEFT JOIN with NULLs

6. **Complex** (Advanced patterns)
   - `revenue_by_region_category`: Multi-dimensional analysis
   - `customer_segmentation`: Complex filtering + aggregation
   - `high_value_customers`: HAVING clause
   - `return_analysis`: CASE WHEN conditionals
   - `quarterly_comparison`: Date functions
   - `profit_analysis`: Percentage calculations
   - `customer_region_spend`: Top N per group

## How to Use

### 1. Generate Sample Data

```bash
python create_sample_data.py
```

This creates all four parquet files in this directory.

### 2. Run Basic Prototype

```bash
# Set up environment
cp .env.example .env
# Edit .env with your OpenAI API key

# Run notebook
jupyter notebook prototype.ipynb
```

### 3. Run Advanced Examples

```bash
jupyter notebook prototype_advanced.ipynb
```

This notebook demonstrates:
- Multi-table joins
- Complex aggregations  
- Conditional aggregation (CASE WHEN)
- GROUP BY with multiple columns
- HAVING clauses
- Multi-turn conversation with context
